Implementa mega menu custom, no Radix NavigationMenu. Quiero control total y cero viewport raro.

1) Estructura del header (HTML)
<nav className="relative">
  <div
    className="group relative inline-block"
    onMouseEnter={() => open("product")}
    onMouseLeave={scheduleClose}
  >
    <button className="nav-pill">Product</button>

    {openMenu === "product" && (
      <div
        className="mega-wrap"
        onMouseEnter={cancelClose}
        onMouseLeave={scheduleClose}
      >
        <div className="mega-panel">
          {/* grid 4 columnas */}
        </div>
      </div>
    )}
  </div>

  {/* repetir para Resources */}

2) Lógica hover EXACTA (delay de cierre para cruzar el gap)
const [openMenu, setOpenMenu] = useState<"" | "product" | "resources">("");
const t = useRef<number | null>(null);

const open = (m: "product" | "resources") => {
  if (t.current) window.clearTimeout(t.current);
  setOpenMenu(m);
};
const scheduleClose = () => {
  if (t.current) window.clearTimeout(t.current);
  t.current = window.setTimeout(() => setOpenMenu(""), 160);
};
const cancelClose = () => {
  if (t.current) window.clearTimeout(t.current);
};

Y cada columna:

Definition of Done

El panel NO se estira a los bordes (siempre 1040px centrado).

Hover no se cierra cuando muevo mouse del trigger al panel.

No hay borde duro; se ve glass.

Hover de items es un “pill” alrededor del contenido.


---

## Por qué esto te quita el dolor de cabeza
Porque **eliminas**:
- `NavigationMenuViewport` (el culpable de full width)
- `motion` raro de Radix
- clases internas que Replit no entiende y rompe

Y te quedas con control total: 1 estado + 1 timer + un div absoluto.
